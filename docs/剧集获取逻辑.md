## 使用`AnitomySharp 剧集解析器`获取剧集

1. 确定系列 ID
   - 使用本地配置的 ID
   - 如果启用`处理多季度目录`
     - 跳过特殊目录
     - 若 Jellyfin 已配置 ID，则跳过
     - 根据剧集的父目录重新获取 ID
2. 基础规则匹配
   - 获取剧集列表，按索引查找对应剧集
   - 若类型为 Special 且列表为空，则改用全部类型重新获取
   - 若索引为 0 且列表非空，返回第一集
   - 若存在替代集数，则尝试用替代集数匹配
3. 多季度规则匹配
   - 如果基础匹配失败且为普通剧集，则尝试处理多季连续编号场景
   - 查找当前系列的续集，累计总集数，定位目标系列和相对索引，再次匹配

```mermaid
graph TD
    A[开始] -->A1[获取系列 ID]
    A1 --> F{启用配置?<br>（处理多季度目录）}
    subgraph G[更新系列 ID]
        direction TB
        G1[获取父目录]-->G2{跳过指定目录}
        G2-->|否|G3{Jellyfin 已配置 ID}
        G2-->|是|G9[返回 ID]
        G3-->|否|G4[搜索目录获取新 ID]
        G3-->|是|G9
        G4-->G9
    end
    F -->|是| G
    F -->|否| I
    G --> I
    subgraph I [基础规则匹配]
        direction TB
        J[获取剧集元数据]-->J1{有数据}
        J1-->|是|J2{匹配到剧集元数据}
        J1-->|否|J3[可能因剧集类型错误<br/>重新获取剧集元数据]
        J3-->J2
        J2-->|是|J4[返回剧集元数据]
        J2-->|否|J5[索引为0时返回第一集]
        J2-->|否|J6[尝试替代集数匹配]
        J5-->J4
        J6-->J4
    end
    I --> L{匹配到剧集？}
    subgraph N [尝试多季度规则]
    end
    L -->|否| N
    L -->|是| P1[处理元数据]
    N --> P{匹配到剧集？}
    P -->|是| P1
    P -->|否| U[创建特典剧集]
    P1 --> Z[返回剧集]
    U --> Z
```
